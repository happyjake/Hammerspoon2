hs.calendar.listEvents(calendar, start, end) -> [[String: Any]]

List Events from one Calendar that overlap a time window. The window may span at most four years.
Declaration
hs.calendar.listEvents(calendar, start, end) -> [[String: Any]]
Parameters
Name Type Description
calendar string Calendar id or exact title
start string Window start as a valid ISO 8601 datetime with an explicit offset from `-14:00` through `+14:00`, or `Z`
end string Window end as a valid ISO 8601 datetime with an explicit offset from `-14:00` through `+14:00`, or `Z`
Returns
[[String: Any]]
Event objects containing `id`, `title`, `start`, `end`, `allDay`, `location`, `notes`, `url`, `attendees`, `organizer`, `status`, `availability`, `recurring`, and `occurrenceStart`. Timed values use UTC `Z`; all-day values are `YYYY-MM-DD`. Each attendee and organizer contains `name`, `url`, `status`, `role`, `type`, and `currentUser`. Non-recurring Events have a `null` `occurrenceStart`.
Example
const events = hs.calendar.listEvents(
  'Work',
  '2026-07-12T00:00:00Z',
  '2026-07-13T00:00:00Z'
)

hs.calendar.createEvent(options) -> [String: Any]

Create a single Event. Timed `start`/`end` values require an explicit UTC offset or `Z`; all-day values must be `YYYY-MM-DD`. `calendar` resolves by id first, then exact title, and defaults to the Calendar for new Events when omitted. `alarms` contains non-negative minutes-before values. Recurring Event creation is unsupported.
Declaration
hs.calendar.createEvent(options) -> [String: Any]
Parameters
Name Type Description
options JSValue Event fields: `title`, `start`, `end`, optional `calendar`, `allDay`, `location`, `notes`, `url`, and `alarms`
Returns
[String: Any]
The created Event as a plain object; invalid input, unavailable targets, and save failures throw a JavaScript `Error`
Example
const event = hs.calendar.createEvent({
  title: 'Project check-in',
  start: '2026-07-13T01:00:00Z',
  end: '2026-07-13T01:30:00Z',
  alarms: [10]
})
console.log(event.id)
